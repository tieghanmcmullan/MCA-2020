Week 5: Basic Metadata

Task 1: Creating a Metadata Scheme

If I were to have 1000 different MEI files in my GitHub repository, these are the metadata elements I would use to describe them as I believe they are the most important:

  1. Metadata about the file itself:

    • Title: name of transcribed piece
    • Composer: name of the individual or band who created the piece
    • Arranger: name of the person or band responsible for the particular arrangement of the piece
    • Source: source which the file was transcribed from
    • Date: date of publication
    • Lyricist: name of the individual or band who wrote the lyrics
    • Encoder: name of the individual(s)that encoded the file
    • Size: size of the file
    • Publisher: name of the individual or organisation responsible for the publication of the piece
    • Publisherâ€™s location: postal address of publisher
    • Usage Restrictions: legal statute of the work regarding its distribution, publication, etc.

  2. Metadata about the encoding process:

    • Project Description: describes the purpose of the file's creation
    • Editorial Changes: described editorial practices which occurred during the transcription of the file

  3. Metadata about the original piece:

    • title: name of original piece
    • arranger: name of the arranger of original piece
    • composer: name of the individual(s) reponsible for the creation of the original piece
    • lyricist: name of the individual(s) who wrote the lyrics for the original piece
    • year: year of the orignal work's creation
