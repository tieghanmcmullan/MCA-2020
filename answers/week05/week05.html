Week 5: Basic Metadata

Task 1: Creating Metadata Schema

If I were to have 1000 different MEI files in my GitHub repository... These are what I believe are the most important metadata elements I would use for description:

  1. Metadata about the file itself:

    • Title: The name of the transcribed piece
    • Composer: The name(s) of the individual(s) who created the piece
    • Arranger: The name(s) of the person(s) responsible for arranging the piece
    • Source: The source from which the file was transcribed
    • Date: Date of publication
    • Lyricist: The name(s) of the individual(s) who wrote the lyrics
    • Encoder: The name(s) of the individual(s) who encoded the file
    • Size: Size of the file
    • Publisher: The name of the individual or organisation responsible for the publication of the piece
    • Publisher location: The address or location of publisher
    • Usage Restrictions: The legalalities of the work's distribution, publication, usage and more.

  2. Metadata about encoding process:

    • Project Description: Description of the purpose of the file's creation
    • Editorial Changes: Description of whatever editorial practices took place during the file's transcription

  3. Metadata about the original piece:

    • Title: The name of original piece
    • Arranger: The name(s) of the arranger(s) of original piece
    • Composer: The name(s) of the individual(s) reponsible for the creation of the original piece
    • Lyricist: The name(s) of the individual(s) who wrote the lyrics for the original piece
    • Year: The year the original work was created
